网上对 Hermes Agent 的讨论大多集中在"怎么问答"这个层面，但它真正被低估的能力是：**当个人助理用**。

前提是要跨过一个门槛——默认的 Hermes 没有外部能力，不会翻你的日历、邮箱或任务列表。一旦接上这些，它就不再是一个等你去问的聊天框，而是一个可以主动帮你干活、记住上下文、定时汇报的助理。

下面是我整理的五步设置框架。

## 一、安装

先把 Hermes 跑起来，两种方式都很快：

- **桌面客户端**：直接下载打开，不需要终端
- **终端安装**：一行脚本完成，适合习惯命令行操作的人

装完不要停在这里。一个默认的 Hermes 没有外部能力，调用的价值很低。这个阶段的唯一目标是把服务跑起来，后续配置在聊天界面里完成会更直观。

## 二、接上 Composio，这是最容易被跳过的一步

这是整套设置里**最关键的一步**，也是大多数人没做的部分。

Composio 的作用是把 Hermes 和外部工具桥接起来。不接的话，Hermes 不仅碰不到你的日历和邮箱，连基本的任务管理都做不到。接上之后，可以直接在 Hermes 里操作：

- Google Calendar / Outlook
- Gmail
- Google Tasks
- Slack
- GitHub
- 其他 1000+ 集成应用

连接方式很简单：在 Hermes 聊天里发送 `composio.dev/hermes`，按提示完成授权即可。

除了工具接入，还有两个开关必须打开：

- **持久记忆（persistent memory）**：让 Hermes 跨会话记住上下文，而不是每次对话从零开始
- **技能生成（skill generation）**：把 Hermes 完成的每次任务自动保存为可复用的工作流，下次直接调用

这两个功能的复利效应很明显——用得越久，Hermes 越像一个了解你习惯、知道怎么帮你省事的助理，而不是每次都要重新交代背景的工具。

## 三、接消息网关，把助理装进口袋

默认的 Hermes 只能通过桌面客户端访问。把它接入一个你已经在日常使用的消息平台，才能真正做到"随时能找到"。

可选平台：

- Discord
- Telegram
- Slack
- WhatsApp
- Signal

接入方式有两种：

- 在 Hermes 聊天里直接说：*"Set up my messaging gateway. I want to connect [platform] so I can reach you from my phone. Walk me through authorizing it."*
- 终端命令：`hermes gateway setup`

网关默认走 allowlist 模式，随机消息发不过来，安全性没问题。

这个入口的便利性直接决定使用频率。手机上是最高频的入口，没有之一。

## 四、像真正的助理一样派活

接好工具和入口之后，使用方式要从"问答模式"切换到"代理模式"。

不是问它问题，而是给它明确的执行任务。我常用的几类：

- *"今天日历上有什么？有没有冲突？我有没有 double booked？"*
- *"把我任务数据库里过期的和未来三天要到期的东西列出来。"*
- *"检查我的 Slack 未读消息和频道，总结那些需要我回复的内容。"*

比单次执行更有价值的是**定时任务**。Hermes 内置了 cron 调度器，可以设置 recurring 任务：

- *"每天早上 9 点扫描 Slack，给我发一份简报。"*
- *"睡前给我发一个待办更新，让我能规划明天的优先级。"*

执行过程中如果发现方向错了，直接发 `/stop` 就能中断并改写指令，不需要等它跑完再手动修正。这个能力在处理半结构化、需要中途纠偏的任务时特别有用。

## 五、个性化配置

基础功能跑通之后，第三步是把它从"通用助理"变成"你的助理"。

可调整的方向：

- **人格/语气**：工作场景用直接务实的风格，私人场景可以更放松
- **模型选择**：不同模型的输出风格差异明显，按任务类型切换
- **主频道**：用 `/sethome` 设一个控制中心，避免对话散落在各个平台
- **动画头像**：`/pet` 可以给 Agent 换 animated avatar，有近 3000 个选项，会根据当前工作状态变化

如果是在团队或多 Agent 场景下使用，不同平台适合不同角色。Discord 最适合做集中 hub，可以建不同频道对应不同子任务；Telegram 和 WhatsApp 适合快速消息，但群组能力不如 Discord。

## 总结

这套方法的核心逻辑是：

1. **先接工具**，让 Agent 具备行动能力
2. **再接入口**，让 Agent 具备可及性
3. **用代理式指令替代问答式交互**
4. **靠记忆和技能形成复利**
5. **最后才是风格上的自定义**

一句话：**不是把 AI 当搜索引擎来问，而是把它当助理来派活**。后者前期配置成本更高，但效果边界完全不同。一旦跑通，你会明显感觉到：这个助理确实不会睡觉，也确实在记住你说的每一件事。
