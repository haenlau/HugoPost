前阵子折腾 AI 做 PPT 踩了不少坑，要么生成后不能改，要么结构是对的、视觉却没法看。

最近试用了一个叫 **dashi-ppt-skill** 的项目，体验比预期好很多。它不是简单的“帮你写大纲”，而是真正输出一份**可编辑、自带控制台、能导出真实 PPTX** 的演示文稿。

项目地址：[chuspeeism/dashi-ppt-skill](https://github.com/chuspeeism/dashi-ppt-skill)

---

## 它是什么

dashi-ppt-skill 是一个专门给 AI Agent 用的 PPT 生成 skill。给它一篇文档，它会自动整理出结构、套版式、填内容、排图表，最终生成一个**网页版 PPT 编辑器**。

在这份产物里，你不是得到一个静态页面，而是得到一套**可以继续编辑**的演示文稿。

---

## 我比较在意的地方

### 1. 生成的 PPT 能继续改

这是最重要的点。很多同类工具只负责“生成”，生成之后内容锁死，想改个数字或换张图只能重来一遍。dashi 的思路不同：生成的是网页版编辑器，文字点一下就能改，图片拖拽替换，页面顺序可以拖放重排。

### 2. 不只是排版，还有分析模型版式

除了常规的封面、目录、趋势图、对比页，它还内置了 SWOT、波特五力、PEST、商业模式画布、双钻模型这些分析框架的版式。做行业研究、竞品分析、项目汇报时，这类版式比纯文字页实用很多。

### 3. 12 套主题，1020 个版式

内置 12 套视觉主题，20 种页面角色。选一套主题之后，后面整份 PPT 的配色、字体、间距都会保持一致。不用每页手动对齐。

### 4. 图表和分析模型

页面不只是图文排版，还支持雷达图、瀑布图、矩形树图、漏斗图、热力图、桑基图、甘特图。对汇报型 PPT 来说，这是刚需。

### 5. 导出真实可编辑的 PPTX

最后能导出成真实 PPTX，文字保持可编辑，不是一张图片拼出来的假 PPT。这比“导出成图片/PDF”能往下接的工作流多很多。

## 怎么用

### 环境要求：Node.js 20+ 和 npm

安装命令：

```bash
npx dashi-ppt-skill@latest
```

国内网络镜像：

```bash
npx --registry=https://registry.npmmirror.com dashi-ppt-skill@latest
```

### 基本流程

1. 把手头的文档或主题丢给 Agent
2. 直接说“做一份 PPT”
3. 它会展示 12 套主题让你选一套
4. 自动组稿，生成网页版 PPT 编辑器
5. 在编辑器里改文字、换图片、调版式
6. 导出成 HTML / PDF / PPTX

### 生成后怎么改

- 点文字直接编辑
- 点或拖拽替换图片、视频
- 右侧控制台调整模块数量、布局、配色
- 左侧缩略图拖拽重排页面
- 顶栏进入放映模式、切换明暗主题

### 导出

```bash
npm --prefix <项目目录> run export:pptx -- <输出目录>/ppt 输出.pptx
npm --prefix <项目目录> run export:pdf -- <输出目录>/ppt
```

导出 PPTX / PDF 需要本机装有 Chrome / Chromium / Edge，可用 `CHROME_PATH` 环境变量指定浏览器路径。

## 注意什么

### 本地运行，内容不传服务器

生成、编辑、导出都在本机完成。首次联网只是为了 npm 安装依赖，之后只有静默版本检查会联网，不上传任何内容。

## 适合谁

如果你经常需要做：

- 行业研究
- 融资复盘
- 竞品分析
- 趋势报告
- 项目汇报
- 方案展示
- 路演材料
- 内部培训

这套方案的产出会比较完整。结构统一、视觉一致、还能继续改，比从零搭一份 PPT 省很多时间。

不适合的场景也很明确：如果要求逐像素手工定制、品牌视觉极重，那这个工具的风格化产出会显得太模板化。

---

## 总结

dashi-ppt-skill 的核心价值不是“自动生成”，而是**生成后的可编辑性**。AI 先把架子搭好，后续的人工调整空间还在，这才是真正能接住工作流的形态。

如果你也在找一份“先出稿、再改稿”的 PPT 方案，可以试试。
